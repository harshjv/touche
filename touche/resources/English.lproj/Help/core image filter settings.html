Core Image filter settings

For some input sources (cameras), Touché offers various filters for preprocessing before a captured frame is passed to down the tracking pipeline. These filters can be helpful in order to get the best tracking results possible for your setup, since they can be used to minimize the effects of adverse factors such as stray IR light, weak blobs due to a not-so-perfect multitouch table or excessive camera noise.

You can adjust the filter settings in the "Core Image Filter Settings" box in the tracking pipeline configuration window, which can be accessed via "Configure Tracking Pipeline..." in the "Setup" menu.

Note that depending on your selected input source, filters may not be available.

Also note that you can set the filter stage from which the preview video stream is taken (see "Blob input method settings" and "Blob input preview window").

Note that the filters are applied in the same order as they appear in the left side of the Core Image filter settings box (from top to bottom), and from left to right in terms of tab groups.

In order to keep the filter setup somewhat tidy, the various filters are distributed into 3 groups: Preprocessing, thresholding and postprocessing operations. For more information about the filters in each particular group, please refer to the following pages.

  • Preprocessing filters
  • Thresholding filters
  • Postprocessing filters


Render filters on CPU

Since getting a processed texture back from the GPU in order to pass it on for processing in the tracking pipeline is a very expensive operation in terms of performance, you can set Touché to compute the filters on the CPU instead. In 99% of cases, you want to have this option enabled.
