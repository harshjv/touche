Camera to screen coordinate conversion settings

The camera to screen coordinate conversion is necessary in order to compute a screen position for each blob detected by the camera. This is done by providing reference points with known screen and camera positions during a calibration process.

The following options are available for the camera to screen coordinate converter:


Amount of calibration points per screen axis

The setting of this configuration item only comes into effect when you do a new calibration process. It specifies the amount of calibration points per axis that will make up the calibration grid.

As a rule of thumb, the larger the disparity between the blobs as they appear on the camera and the screen, the more calibration points should be used. Usually, between 3 and 8 points per axis yield very good results, although more points can be necessary in some cases. If the camera view resembles the screen closely, 2 points per axis might also suffice.

Note that large amounts of calibration points can make the coordinate conversion process slower during normal operation, although this effect should be negligible.


Transform blob bounding boxes

If this checkbox is on, the converter will also transform the bounding box of each touch to screen dimensions. You can turn this off if you are not interested in using the bounding boxes, which will yield better performance.


Transform blob edge vertices

If this item is set, the converter will transform the vertices of the detected blob edges to screen dimensions. If you are not intending to use the edge vertices, you can turn this off in order to enhance performance.
