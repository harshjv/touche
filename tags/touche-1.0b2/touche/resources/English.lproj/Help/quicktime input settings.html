Quicktime input settings

If QuickTime is selected as the input method, you can choose any currently connected QuickTime-compatible camera as a capture source. For alternative input methods, see "Blob input method settings".

The following settings are available:


QuickTime Capture Device

This menu is used to choose the QuickTime-compatible video device that should be used for capturing. If you have multiple cameras connected, be sure to select the one you want to use here.

If the selected camera is not available when Touché starts the tracking pipeline in the future, it will automatically fall back to the default video device. If no video device is connected at all, an error will be reported.


Capturing Resolution

You can choose the resolution of frames captured by the camera here. Larger settings can provide for better tracking accuracy, but tend to be (a lot) slower than smaller settings. 320x240 is a good default choice for older Macs, whereas powerful Macs should also be doing fine with higher settings.

Note that the resolutions selectable here do not necessarily correspond to the resolutions that your camera supports natively. QuickTime will automatically resample each frame to the selected size if the camera does not support the resolution natively.

If you want to use your camera's native resolution, please have a look at your camera's manual in order to find out what the native resolution is.


Maximum FPS

Touché will try to process frames as soon as they are delivered by the camera, dropping frames only if it can't keep up with the camera's pace. Thusly, cameras with high framerates might cause Touché to use up most of the available CPU time, leaving little for the actual multitouch apps running next to the Touché tracker.

Therefore, you can set a cap on the framerate. If the camera delivers frames at faster rates than this setting, Touché will start dropping frames in order to process no more frames per second than what you have selected here, saving CPU time.


Minimum blob diameter

You can specify a minimum size for the blobs that should be recognized and tracked. Any blobs that have a diameter below this threshold will be disregarded. This is useful in order to ignore stray white pixels or very small blobs that you do not intend to track.

Note that this setting is given in percentage of the capture resolution in order to be independent from the actual resolution. As an example, if you have selected 1% here, Touché will only track blobs with diameters above 3.2 for 320x240, above 6.4 for 640x480, etc.


Enable blob tracking

If this checkbox is unchecked, Touché will not deliver the frames to the tracker, but will still capture video data and apply filters. This is useful during the setup process, where you do not want to track blobs yet, while you are still fiddling with the filter settings.

Be sure to have this checkbox checked before starting a multitouch app.
