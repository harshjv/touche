libdc1394 input settings

libdc1394 is a high-performance library for capturing video data from 1394 (FireWire) cameras. It is compatible with all FireWire cameras that support the IIDC specification, such as Apple's original external iSight.

Additionally, libdc1394 allows you to fiddle with additional parameters that QuickTime does not expose, such as focus or brightness settings.

The following settings are available:


libdc1394 Camera

This menu is used to choose the IIDC-compatible video device that should be used for capturing. If you have multiple cameras connected, be sure to select the one you want to use here.

If the selected camera is not available when Touché starts the tracking pipeline in the future, it will automatically fall back to a default. If no video device compatible with libdc1394 is connected at all, an error will be reported.


Capturing Resolution

You can choose the resolution of frames captured by the camera here. Larger settings can provide for better tracking accuracy, but tend to be (a lot) slower than smaller settings. 320x240 is a good default choice for older Macs, whereas powerful Macs should also be doing fine with higher settings.

Note that you can only select resolutions that your camera supports natively. If you want to use a resolution other than the ones supported by your camera, you will have to use a QuickTime input source instead, which can resample captured frames to whichever resolution you want.


Maximum FPS

Touché will try to process frames as soon as they are delivered by the camera, dropping frames only if it can't keep up with the camera's pace. Thusly, cameras with high framerates might cause Touché to use up most of the available CPU time, leaving little for the actual multitouch apps running next to the Touché tracker.

Therefore, you can set a cap on the framerate. If the camera delivers frames at faster rates than this setting, Touché will start dropping frames in order to process no more frames per second than what you have selected here, saving CPU time.


Minimum blob diameter

You can specify a minimum size for the blobs that should be recognized and tracked. Any blobs that have a diameter below this threshold will be disregarded. This is useful in order to ignore stray white pixels or very small blobs that you do not intend to track.

Note that this setting is given in percentage of the capture resolution in order to be independent from the actual resolution. As an example, if you have selected 1% here, Touché will only track blobs with diameters above 3.2 for 320x240, above 6.4 for 640x480, etc.


Enable blob tracking

If this checkbox is unchecked, Touché will not deliver the frames to the tracker, but will still capture video data and apply filters. This is useful during the setup process, where you do not want to track blobs yet, while you are still fiddling with the filter settings.

Be sure to have this checkbox checked before starting a multitouch app.


Focus, Shutter, Gain, Brightness, Exposure

With libdc1394, these camera parameters can be manipulated. Note that not all of these features might be supported by your camera. If a feature is not supported to be manipulated, the corresponding interface elements will be disabled.

Additionally, some cameras support an "Auto" mode for certain features. For example, Apple's original external iSight supports an "Auto" mode for the focus setting. Note that if a camera doesn't support an "Auto" mode for a feature, the checkbox will be disabled. Checking the box will set the feature to "Auto" mode, whereas unchecking the box will return the feature to manual control.

Note that some features might not have a visible effect on the camera image (This is due to the camera claiming to support the features when it actually doesn't).
