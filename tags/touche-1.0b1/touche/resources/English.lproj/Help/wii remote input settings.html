Wii Remote input settings

You can use a Wii Remote as an infrared camera with Touché by selecting "Wii Remote" as blob input source. Note that your Mac needs to support Bluetooth for this feature, and that Bluetooth has to be activated.

Once you have selected "Wii Remote" input or once you start Touché with the Wii Remote input source, you will be prompted to hold down the 1 and 2 buttons simultaneously on the Wii Remote you wan't to connect. Keep these buttons held down until Touché has connected to the Wii Remote.

While trying to connect a Wii Remote, be sure not to have a running Wii console nearby, since it will jump onto the Wii Remote's signal faster than Touché can and will thusly prevent Touché from connecting to the Wii Remote.

Note that the Wii Remote's IR tracking mechanism can only track up to 4 touches at the same time. If you want to use more touch points, you will have to use a camera source instead (see "Tracking pipeline configuration").

The following settings are available for the Wii Remote input source:


Maximum FPS

Touché will try to process IR data as soon as it is delivered by the Wii Remote, dropping data only if it can't keep up with the Wii Remote's pace. Since the Wii Remote has a very high framerate, it might cause Touché to use up most of the available CPU time to process IR data, leaving little for the actual multitouch apps running next to the Touché tracker.

Therefore, you can set a cap on the framerate. If the Wii Remote delivers data at faster rates than this setting, Touché will start dropping data in order to process no more frames per second than what you have selected here, saving CPU time.


Enable blob tracking

If this checkbox is unchecked, Touché will not deliver the frames to the tracker, but will still capture IR data notmally. This is useful during the setup process, where you do not want to track blobs yet, while you are still fiddling with the other settings.

Be sure to have this checkbox checked before starting a multitouch app.
